# Demo-пакет данных

Синтетический мини-набор для демонстрации сценария: **реестр позиций → PDF техдокументации → карточка атрибутов**.

Без реальных данных заказчика. Классы: электродвигатель (`demo-motor`), насос (`demo-pump`), арматура (`demo-valve`).

## Что внутри

| Путь | Назначение |
| --- | --- |
| `registry.json` | Реестр: какая позиция → какие PDF |
| `pdf/` | ТУ / паспорт / скан без текстового слоя |
| `processed/` | Справочник атрибутов класса и эталонные значения |
| `build_demo_pdfs.py` | Пересборка PDF (ReportLab) + `apps/demo-app/pdf-data.js` |

## Позиции

1. ТУ с титулом, содержанием и **разнесёнными** таблицами (электрика / механика / справочное).
2. ТУ с несколькими вариантами исполнения (нужно выбрать колонку Исп. B).
3. Пара «ТУ + паспорт»: часть значений только в паспорте; в ТУ есть уточняющая сноска по КПД.
4. Скан (изображение без text layer) — OCR-сценарий.
5. ТУ насоса: титул + prose (подача/напор) + конструкция + раздельные таблицы привода/КПД.
6. Паспорт затвора: титул + prose (DN/PN/температура) + таблицы конструкции и ζ.

## Пересборка PDF

```bash
uv run --with reportlab python research/datasets/demo/build_demo_pdfs.py
```

`SCAN-DEMO-004.pdf` скрипт не трогает.

## Как смотреть

Откройте PDF в `pdf/` и сверьте значения с `processed/ground_truth.jsonl`.  
Для demo-app: seed в `apps/demo-app/app.js` + `pdf-data.js`.
